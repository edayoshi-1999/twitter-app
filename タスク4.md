### ステップ3: Tailwind CSS のセットアップ

#### 何をするか？

Tailwind CSSを初期化し、プロジェクトで使えるように設定します。

#### なぜTailwind CSS？

- **開発速度**: クラス名を組み合わせるだけでスタイリング完了
- **一貫性**: デザインシステムが自動的に統一される
- **カスタマイズ性**: 設定ファイルで柔軟にカスタマイズ可能
- **最適化**: 使用していないスタイルは自動削除（PurgeCSS）

#### セットアップ手順

**1. Tailwind CSSの初期化**

```bash
npx tailwindcss init -p
```

これにより以下のファイルが作成されます：
- `tailwind.config.js`: Tailwind の設定
- `postcss.config.js`: PostCSS の設定

**2. `tailwind.config.js` の編集**

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",  // すべてのソースファイルを監視
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

**`content`フィールドの意味**:
- Tailwindがクラス名をスキャンする対象ファイルを指定
- 使用されているクラスのみがビルド時に含まれる（ファイルサイズ削減）

**3. グローバルCSSにTailwindディレクティブを追加**

`src/index.css`:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

**各ディレクティブの役割**:
- `@tailwind base`: リセットCSS、基本スタイル
- `@tailwind components`: 再利用可能なコンポーネントクラス
- `@tailwind utilities`: ユーティリティクラス（`p-4`, `text-center`など）

**4. `main.tsx`でCSSをインポート**

```tsx
import './index.css'
```

#### 動作確認

`App.tsx`で試してみる：

```tsx
function App() {
  return (
    <div className="min-h-screen bg-gray-100 flex items-center justify-center">
      <h1 className="text-4xl font-bold text-blue-600">
        Hello Tailwind CSS!
      </h1>
    </div>

}
```

| **速度** | 普通 | 🚀 **超高速**（Rust製） |
| **設定** | 2つのツールを別々に設定 | ✅ 1つの設定ファイル |
| **統合性** | 競合する場合がある | ✅ 完全に統合 |
| **保守性** | 2つのツールを管理 | ✅ 1つのツールだけ |

```bash
npx @biomejs/biome init
```

### ステップ5: Vitest のセットアップ

#### 何をするか？

**Vitest**（Viteと統合されたテストフレームワーク）を設定します。

#### なぜVitest？

- ✅ **Viteとの完全統合**: 設定ファイルを共有、高速
- ✅ **Jest互換API**: Jestと同じ書き方ができる
- ✅ **高速実行**: Viteの高速なビルドを活用
- ✅ **TypeScript完全サポート**: 型安全なテストが書ける

#### セットアップ手順

**1. `vite.config.ts` の更新**

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

// https://vite.dev/config/
export default defineConfig({
  plugins: [react()],
  test: {
    globals: true,
    environment: 'jsdom',
    setupFiles: './src/test/setup.ts',
    css: true,
  },
})
```

**設定の意味**:
- `globals: true`: `describe`, `it`, `expect`をグローバルに使える
- `environment: 'jsdom'`: ブラウザ環境をシミュレート
- `setupFiles`: テスト前に実行するファイル
- `css: true`: CSSのインポートを許可

**2. テストセットアップファイルの作成**

**3. TypeScript設定の更新**

### ステップ6: React Testing Library のセットアップ

#### 何をするか？

React Testing Libraryの設定とユーティリティを準備します。

#### なぜReact Testing Library？

**従来のテスト（Enzyme など）**:
```tsx
// コンポーネントの内部実装に依存
wrapper.find('button').prop('onClick')()
```

**React Testing Library（推奨）**:
```tsx
// ユーザーの視点でテスト
const button = screen.getByRole('button', { name: 'Submit' })
await userEvent.click(button)
```

**メリット**:
- ✅ ユーザーが実際に操作する方法でテスト
- ✅ 内部実装の変更に強い（リファクタリングしてもテストが壊れにくい）
- ✅ アクセシビリティを考慮したテストが自然に書ける

#### テストユーティリティの作成

`src/test/utils.tsx`:
```tsx
import { render, RenderOptions } from '@testing-library/react';
import { ReactElement } from 'react';

// カスタムレンダー関数（将来的にProviderを追加可能）
function customRender(
  ui: ReactElement,
  options?: Omit<RenderOptions, 'wrapper'>,
) {
  return render(ui, { ...options });
}

export * from '@testing-library/react';
export { customRender as render };
```

**使い方**:
```tsx
import { render, screen } from '@/test/utils';  // カスタムrender
```

### ステップ7: Axios のセットアップ

#### 何をするか？

**Axios**（HTTPクライアント）の基本設定とインスタンスを作成します。

#### なぜAxios？

**ブラウザ標準のfetchとの比較**:

| 機能 | fetch | Axios |
|------|-------|-------|
| **JSONの自動変換** | ❌ 手動で`.json()` | ✅ 自動 |
| **エラーハンドリング** | ❌ HTTPエラーでもresolve | ✅ HTTPエラーでreject |
| **インターセプター** | ❌ なし | ✅ あり（認証トークン追加など） |
| **タイムアウト設定** | ❌ AbortController必要 | ✅ 簡単に設定可能 |
| **リクエストキャンセル** | △ 可能だが複雑 | ✅ 簡単 |

#### Axiosインスタンスの作成

`src/services/api.ts`:
```typescript
import axios from 'axios';

// Axiosインスタンスを作成
export const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
  withCredentials: true, // Cookie を送信（Sanctum認証で必要）
});

// リクエストインターセプター（将来的に認証トークンを追加）
api.interceptors.request.use(
  (config) => {
    // 将来的にトークンをヘッダーに追加
    // const token = localStorage.getItem('token');
    // if (token) {
    //   config.headers.Authorization = `Bearer ${token}`;
    // }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  },
);

// レスポンスインターセプター（エラーハンドリング）
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response) {
      // サーバーがエラーレスポンスを返した
      console.error('API Error:', error.response.status, error.response.data);
    } else if (error.request) {
      // リクエストは送信されたがレスポンスがない
      console.error('Network Error:', error.request);
    } else {
      // リクエスト設定中にエラー
      console.error('Error:', error.message);
    }
    return Promise.reject(error);
  },
);
```

**設定のポイント**:
- **baseURL**: 環境変数から読み込み、デフォルトはlocalhost
- **timeout**: 10秒でタイムアウト
- **withCredentials**: Laravel Sanctum の Cookie 認証で必須
- **インターセプター**: リクエスト/レスポンスを一括処理

#### 環境変数の設定

`.env`:
```env
VITE_API_BASE_URL=http://localhost/api
```

#### APIサービスの作成例

`src/services/tweetService.ts`:
```typescript
import { api } from './api';

export interface Tweet {
  id: string;
  body: string;
  user: {
    id: number;
    name: string;
    username: string;
  };
  likes_count: number;
  created_at: string;
  updated_at: string;
}

export const tweetService = {
  // ツイート一覧取得
  async getTweets(): Promise<Tweet[]> {
    const response = await api.get<{ data: Tweet[] }>('/tweets');
    return response.data.data;
  },

  // ツイート投稿
  async createTweet(body: string): Promise<Tweet> {
    const response = await api.post<{ data: Tweet }>('/tweets', { body });
    return response.data.data;
  },

  // ツイート削除
  async deleteTweet(id: string): Promise<void> {
    await api.delete(`/tweets/${id}`);
  },
};
```

```tsx
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'danger';

export function Button({
  variant = 'primary',
  size = 'md',
  isLoading = false,
  children,
  className = '',
  disabled,
  ...props
}: ButtonProps) {
  const baseStyles = 'font-semibold rounded-lg transition-colors disabled:opacity-50';

  const variantStyles = {
    primary: 'bg-blue-500 text-white hover:bg-blue-600',
    secondary: 'bg-gray-200 text-gray-800 hover:bg-gray-300',
    danger: 'bg-red-500 text-white hover:bg-red-600',
  };

  const sizeStyles = {
    sm: 'px-3 py-1.5 text-sm',
    md: 'px-4 py-2 text-base',
    lg: 'px-6 py-3 text-lg',
  };

  return (
    <button
      className={`${baseStyles} ${variantStyles[variant]} ${sizeStyles[size]} ${className}`}
      disabled={disabled || isLoading}
      {...props}
    >
      {isLoading ? 'Loading...' : children}
    </button>
  );
}
```

## ❓ よくある質問（FAQ）

### Q1: なぜViteを使うのですか？

**A**: Viteは次世代のビルドツールで、以下の理由から選ばれています：
- **高速な開発サーバー起動**: Webpackより圧倒的に速い
- **即座のHMR**: ファイル変更が瞬時に反映
- **シンプルな設定**: ゼロコンフィグで使える
- **最適化されたビルド**: Rollupベースで高速

### Q2: BiomeとESLint + Prettierの違いは？

**A**:

**ESLint + Prettier（従来）**:
- 2つのツールを管理
- 設定が複雑になりがち
- 実行速度が遅い

**Biome（新しい選択肢）**:
- 1つのツールで完結
- 設定がシンプル
- Rust製で超高速

このプロジェクトではBiomeを採用し、開発体験を向上させています。

### Q3: TypeScriptは必須ですか？

**A**: 必須ではありませんが、強く推奨します。

**TypeScriptのメリット**:
- ✅ バグの早期発見（コンパイル時に型エラーを検出）
- ✅ 優れたIDEサポート（自動補完、リファクタリング）
- ✅ ドキュメントとしての役割（型定義がコードの説明になる）
- ✅ 大規模開発に適している

**デメリット**:
- ❌ 学習コストが若干高い
- ❌ ビルド時間が少し長くなる

小規模プロジェクトでもTypeScriptを使うことで、長期的に保守しやすくなります。

### Q4: Tailwind CSSは使いにくくないですか？

**A**: 最初は慣れが必要ですが、慣れると非常に効率的です。

**従来のCSS**:
```tsx
// CSS
.button-primary {
  background-color: blue;
  color: white;
  padding: 0.5rem 1rem;
  border-radius: 0.25rem;
}

// JSX
<button className="button-primary">Click</button>
```

**Tailwind CSS**:
```tsx
<button className="bg-blue-500 text-white px-4 py-2 rounded">
  Click
</button>
```

**メリット**:
- クラス名を考える必要がない
- CSSファイルを行き来しなくて良い
- デザインシステムが自動的に統一される

### Q5: Axiosは必須ですか？fetchじゃダメですか？

**A**: fetchでも可能ですが、Axiosの方が便利です。

**fetch（標準）**:
```typescript
try {
  const response = await fetch('/api/tweets');
  if (!response.ok) {
    throw new Error('Network response was not ok');
  }
  const data = await response.json();
} catch (error) {
  console.error(error);
}
```

**Axios（推奨）**:
```typescript
try {
  const { data } = await api.get('/tweets');
} catch (error) {
  console.error(error);
}
```

Axiosの方がコードが簡潔で、エラーハンドリングも楽です。
